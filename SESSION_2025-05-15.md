## Hauptthema: onDidChangeConfiguration Logic

**Problem**: Unterscheidung zwischen Global und Workspace Settings ist nicht trivial.

### Die "zwei Spaces" Metapher (Stefan)

Es gibt zwei völlig unterschiedliche Konfigurationssysteme:

1. **Space 1 — Klaus'C0dehelfer's Konfiguration (VSCode Settings)**
   - `claude-workspace-monitor.awarenessMode` Einstellung
   - Wo: VSCode's eigenes Settings-System (`~/.config/Code/User/settings.json`, `.vscode/settings.json`, etc.)
   - Wie ablesen: `config.inspect<T>(section)` gibt alle Level-Werte zurück

2. **Space 2 — Claude Code's Hook-Konfiguration**
   - Hook wird in `.claude/settings.local.json` gespeichert
   - Wo: Global (User-gewählter `.claude`-Ordner) oder Workspace (`${workspace}/.claude/settings.local.json`)
   - Basierend auf Space-1-Kontext entscheiden wir, wo zu schreiben ist

### Das Event-Problem

`event.affectsConfiguration(section, scope)` gibt einen boolean zurück, aber:

- `affectsConfiguration(section, workspaceFolder)` ist **IMMER true**, auch bei Global-Änderungen
  - **Grund**: Globale Settings gelten für Workspace (Hierarchie: Default → Global → Workspace → Folder)
- Man kann damit Global nicht direkt ausschließen

**Lösung**: Kombination aus Event + `config.inspect()`:

```
affectsConfiguration(section, workspaceFolder) = true  +  workspaceValue === undefined  →  Global
affectsConfiguration(section, workspaceFolder) = true  +  workspaceValue !== undefined  →  Workspace
affectsConfiguration(section, workspaceFolder) = false (für all folders)  →  Global (oder Subfolder ignorieren)
```

### Die korrekte Logik
```typescript
handleAwarenessChange(event):
  if NOT event.affectsConfiguration('awarenessMode'): return

  config = getConfiguration('claude-workspace-monitor', workspaceFolders[0].uri)
  insp   = config.inspect<string>('awarenessMode')

  isGlobal = false
  primaryFound = false

  // Iteriere über alle Workspaces
  for folder in workspaceFolders:
    if event.affectsConfiguration('awarenessMode', folder):
      if folder === workspaceFolders[0]:           // Primary?
        primaryFound = true
        if insp.workspaceValue === undefined:
          isGlobal = true                          // Kein Override → Global
        break
      else:                                         // Subfolder
        MessageBox: "Subfolder-Konfiguration nicht unterstützt"
        return                                      // Ignorieren

  if NOT primaryFound:
    isGlobal = true                                 // Kein Folder → Global

  // Einmalige Auswertung
  if isGlobal:
    mode = insp.globalValue
    → Dialog für globalen .claude-Ordner
    → oder löschen wenn mode === 'none'
  else:
    mode = insp.workspaceValue
    → schreibe ${workspace}/.claude/settings.local.json
    → oder löschen wenn mode === 'none'

  if mode === 'realTime': "coming soon" MessageBox
```
